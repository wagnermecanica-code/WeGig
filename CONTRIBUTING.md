# Guia de Contribuição

## Como Contribuir

### Antes de Começar
1. Certifique-se de ter a versão mais recente do projeto:
   ```bash
   git pull origin main
   ```

2. Crie uma nova branch para sua funcionalidade ou correção:
   ```bash
   git checkout -b nome-da-sua-branch
   ```

### Durante o Desenvolvimento
1. Faça commits pequenos e frequentes
2. Use mensagens de commit claras e descritivas
3. Teste suas alterações antes de fazer commit

### Padrões de Commit
Use mensagens de commit no seguinte formato:
- `feat: adiciona nova funcionalidade`
- `fix: corrige bug específico`
- `docs: atualiza documentação`
- `style: alterações de formatação`
- `refactor: refatoração de código`
- `test: adiciona ou atualiza testes`
- `chore: tarefas de manutenção`

### Exemplo de Mensagens
- ✅ Bom: "feat: adiciona página de login com validação"
- ✅ Bom: "fix: corrige erro ao salvar formulário"
- ❌ Ruim: "alterações"
- ❌ Ruim: "WIP"

### Finalizando
1. Certifique-se de que tudo está funcionando
2. Faça o push da sua branch:
   ```bash
   git push origin nome-da-sua-branch
   ```

3. Abra um Pull Request no GitHub (se aplicável)

## Dúvidas?
Em caso de dúvidas, entre em contato com a equipe do projeto.
